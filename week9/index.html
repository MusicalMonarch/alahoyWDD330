Chapter 9: The Window Object

The global object in a browser environment is the window object, a representation of the browser window that contains a web page.

  • BOM only makes sense in a browser environment
    • JS can be run in different environments, so environments like Node.js wouldn't have a window object
    • Such environments would still have a global object
  • In a browser environment, the global object is the window object
    • Any global variable (declared without using const, let, or var) created is a property of the window object
    • In general, you can use a global variable without "window." unless you're checking is the global variable has been defined
    • Some predefined methods (like parseInt() and isNaN()) are are methods of the global object
    • Dialog methods are also included in this list of methods
      • window.alert(), .confirm(), and .prompt() are dialogs
      • Note that these should be used with caution since they stop the execution of a program and are also unreliable as they can be disabled from appearing
  • The window object has a number of properties and methods that provide info about the user's browser
    • The navigator property returns a reference to the Navigator object
      • Navigator contains info about the browser being used while its userAgent will return information about the browser and OS being used
    • window.location is an object that contains info about the URL of the current page
      • href would return the full URL as a string
      • protocol returns a string describing the protocol ("https:" or "http:")
      • host returns a string of the domain of the current URL and the port number (often omitted if default port is 80)
        • window.location.host << "www.sitepoint.com"
        • location.hostname would return only the domain while location.port would return a port number (an empty string if port is not stated in the URL)
      • pathname returns a string of the path that follows the domain
      • And there are more properties and methods that location can use
    • history is a property that can be used to access info about previously visited paged in the current browser session
    • screen contains info about the screen the browser is displayed on, including width and height
  • Down the tree of window objects is the Document object
    • One method of the document object is write(); this has been discouraged since it mixes HTML and JavaScript
    • You can use cookies with the document object
      • Cookies can be used to personalize a user's experience on a website
  • window also has some useful methods for scheduling the execution of a function and for repeatedly executing functions at regular intervals
    • setTimeout() accepts a callback to a function (first parameter) and a number of milliseconds (second parameter)
      • window.setTimeout( () => alert("Time's up!"), 3000);
      • This method returns an integer that is an ID and can be used to reference a particular timeout; this integer can then be used with window.clearTimeout()
    • window.clearTimeout(int);
    • setInterval() will repeatedly invoke the callback function after every given number of milliseconds
      • To stop this, use clearInterval() with repeat as an argument because the setInterval method returns an ID that is assigned to the variable repeat
    • Be careful of using "this" with timing methods because "this" will be bound to the window object
