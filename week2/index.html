  • Any variable declared outside of a block has a global scope
    • Likewise, variables declared in blocks (only when using "let" or "const") have a local scope
    • Good programmers limit the amount of global variables
  • Template literals are special types of strings that were introduced in ES6
    • Use the backtick (`) character to deliminate the string
    • Allows interpolation of JavaScript code
      • ex: const name = `Robert`; `Hello, ${ name }!`;
    • Contains line breaks that are all converted into a Line Feed(\n)
    • If you want to place a backtick inside a template literal, it needs to be escaped with a backslash
      • ex: `Here's how you show a \`, which is a backtick`;
    • "Super Strings"
      • Not uncommon to see template literals making all strings
  • Symbols were introduced as a new primitive value in ES6
    • Only way to create them is the Symbol() function
      • ex: const uniqueID = Symbol();
    • The description acts as a string representation of the symbol
    • It's possible for two variables to point to the same symbol if for() is used
      • ex: const A = Symbol.for('shared symbol'); const B = Symbol.for('shared symbol')
  • Sets were introduced in ES6
    • A data structure that represents a collection of unique values
    • No duplicates
    • A useful way to keep track of data without having to check if any values have been duplicated
